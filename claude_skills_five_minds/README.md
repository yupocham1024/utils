# claude_skills_five_minds

Claude 用の Skill。「戦略家・設計者・生成者・破壊者・仕上げ屋」の独立した5つの役割を順に実行させ、単発生成よりも大幅に質の高い成果物を作るワークフローを型として組み込む。[Claude Code](https://claude.com/claude-code)・[claude.ai](https://claude.ai) の両方で導入できる(挙動に一部差分あり。後述)。

元ネタ: [Claude Sonnet 極限起動ワークフロー](https://youmind.com/ja-JP/landing/x-viral-articles/claude-sonnet-extreme-activation-workflow)

## これは何をするか

「5人体制で」「/five-minds」などと**明示的に呼び出した時だけ**、Claude が以下のワークフローを実行する(通常の依頼には自動発火しない)。

1. **戦略家** — いきなり作らず、まず勝利条件と採点基準(10点満点の採点軸)を定義する。ターゲット・欲望・反論ポイント・勝ち筋を分析する
2. **設計者** — 完成までの工程分解、構成案、必要情報、失敗リスクを整理する
3. **生成者** — 成果物を必ず3案(素直な案・尖った案・攻めた案)作り、強み・弱み・リスクを比較して1案を選ぶ
4. **破壊者** — 褒めることを禁止された批評家として、採点基準に照らして弱点だけを徹底的に指摘する
5. **仕上げ屋** — 指摘をすべて処置(修正 or 理由付きで意図的に残す)し、公開可能な完成度に仕上げる

全役割が毎回「判断・理由・不足情報・失敗リスク・改善案・次の行動・自己採点」を出力する。

このサイクルを成果物のサイズに応じて回す。

- 大きな成果物(記事・動画スクリプト・長文ドキュメント・複数ファイルのコード): フル3周(方向性決定 → 弱点破壊 → 完成)
- 小さな成果物: 1周 + 破壊→仕上げの短縮2周目
- 破壊者の採点が9/10以上かつ致命的指摘ゼロになったら早期終了

詳細な役割定義・入出力の受け渡しルールは [`five-minds/SKILL.md`](five-minds/SKILL.md) を参照。

## 「独立した5人」の実現方法

役割ごとに**入力ドキュメントを厳格に制限する**ことで独立性を作っている。特に破壊者には勝利条件書と成果物しか渡さず、生成者の選定理由や言い訳を見せない(事情を汲んだ甘い批評を防ぐため)。

- **Claude Code**: 生成者・破壊者を別々のサブエージェント(独立したコンテキストを持つ別インスタンス)として起動し、本当の意味でコンテキストを分離する
- **claude.ai**: サブエージェント機能がないため、1つの応答内で役割を切り替えるロールプレイで代替する。役割ごとの入力制限ルールは同じものを適用する

## 使い方

```
/five-minds で、WordPressの独自ブロック開発の入門記事を書いて
```

```
5人体制で、対数の定義を説明する動画スクリプトを作って
```

呼び出すと、まず戦略家が勝利条件と採点基準を提示するので、方向性が違えばその時点で修正できる。あとは各役割の進捗報告を挟みながら完成版まで自動で進む。

## インストール方法(Claude Code)

Claude Code の Skill は `~/.claude/skills/` 配下に置くと、ユーザーが使う **すべてのプロジェクトで** 有効になる。

### 1. `five-minds` フォルダをコピーする

```bash
cp -r five-minds ~/.claude/skills/five-minds
```

このリポジトリを直接 clone している場合は symlink でもよい。

```bash
ln -s "$(pwd)/five-minds" ~/.claude/skills/five-minds
```

### 2. Claude Code を再起動する(または新しいセッションを開始する)

Skill 一覧はセッション開始時に読み込まれるため、新しいセッションを開始することで `five-minds` が Skill 一覧に表示されるようになる。

### 3. 動作確認

新しいセッションで明示的に呼び出してみる。

```
5人体制で、○○についての短い紹介文を書いて
```

戦略家の「勝利条件・採点基準」から始まれば導入成功。逆に「○○についての紹介文を書いて」とだけ言った場合には発火しないことも確認するとよい。

### アンインストール

```bash
rm -rf ~/.claude/skills/five-minds
```

(symlink で導入した場合は `rm ~/.claude/skills/five-minds`)

## インストール方法(claude.ai)

claude.ai では Skill を **ZIP ファイル** としてアップロードする。Claude Code とはアップロード先が別なので、両方で使う場合は別途アップロードが必要。

### 前提条件

- **コード実行(Code execution)機能が有効になっていること**。無効の場合は Settings 内でオンにする。
- Free / Pro / Max / Team / Enterprise のいずれのプランでも利用可能。

### 1. `five-minds` フォルダを ZIP 化する

`SKILL.md` がフォルダ直下(ルート)に来るように zip 化する。

```bash
cd claude_skills_five_minds
zip -r five-minds.zip five-minds
```

### 2. claude.ai にアップロードする

**Settings > Features > Skills** から、作成した `five-minds.zip` をアップロードする。

### 3. 動作確認

新しいチャットで「5人体制で〜を作って」と依頼し、戦略家の勝利条件定義から始まれば導入成功。

## claude.ai 特有の注意点

- サブエージェントが使えないため、5役割は1応答内のロールプレイになる。コンテキストの物理的な分離はされないので、破壊者の批評の「初見度」は Claude Code 版よりやや弱くなる
- フル3周は応答が長くなるため、周回の区切りで応答が分かれることがある。「続けて」で次の周回に進む

## 注意事項

- このワークフローは通常の生成の数倍のトークンと時間を使う。品質を上げたい成果物にだけ明示的に使うこと
- Skill は「Claude 自身が発火条件を判断する」仕組みのため、100% 確実な発火/非発火は保証されない。誤発火する場合は `five-minds/SKILL.md` の description の SKIP 条件を強める
- prompt-improver(同リポジトリの別 Skill)とは役割が異なり併存できる。prompt-improver は「依頼文の曖昧さの解消」、five-minds は「生成工程の多役割分離」を担当する
- Claude Code と claude.ai でカスタム Skill は自動的に同期されない。両方で使いたい場合はそれぞれに導入する必要がある
